# Contoh Jawaban Essay

**41. Landing page adalah halaman web yang dibuat untuk tujuan tertentu, seperti promosi produk, pendaftaran, pembelian, atau pengumpulan data pengguna. Landing page biasanya memiliki satu fokus utama agar pengguna terdorong melakukan aksi tertentu. Struktur utamanya meliputi hero section atau summary, call to action, social proof, benefits, features, dan repeated CTA. Hero section digunakan untuk menarik perhatian awal, CTA untuk mengarahkan aksi, social proof untuk membangun kepercayaan, benefits untuk menjelaskan manfaat, features untuk menjelaskan fitur, dan repeated CTA untuk meningkatkan peluang konversi.**

**42. Horizontal scrolling adalah pengguliran ke samping, biasanya digunakan untuk carousel, card kategori, galeri gambar, atau pilihan produk. Vertical scrolling adalah pengguliran dari atas ke bawah, biasanya digunakan pada landing page panjang, feed, detail produk, atau halaman mobile. Gabungan horizontal dan vertical scrolling digunakan ketika konten perlu digeser ke berbagai arah, misalnya peta, diagram besar, atau visualisasi data yang ukurannya lebih besar dari frame.**

**43. Trigger adalah pemicu interaksi, misalnya On Click, On Drag, Hover, atau Mouse Enter. Action adalah aksi yang terjadi setelah trigger dilakukan, misalnya Navigate To, Scroll To, Open Overlay, atau Change To. Destination adalah frame atau tujuan akhir dari interaksi tersebut. Animation adalah pengaturan efek perpindahan, seperti Instant, Dissolve, Smart Animate, Move In, Push, Slide In, beserta direction, easing, dan duration.**

**44. Component penting karena membantu desainer membuat elemen reusable yang bisa digunakan berulang, seperti button, card, form, input, navbar, atau icon. Jika elemen dibuat sebagai component, perubahan cukup dilakukan pada main component, lalu instance yang terhubung akan ikut berubah. Ini membuat desain lebih konsisten, efisien, mudah diperbarui, dan cocok untuk project besar. Tanpa component, desainer harus mengedit elemen satu per satu sehingga memakan waktu dan rawan tidak konsisten.**

**45. Alurnya dimulai dari membuat Colors atau Primitives, yaitu kumpulan warna dasar yang menyimpan nilai asli seperti HEX, misalnya Grey/50 atau Grey/900. Setelah itu dibuat Tokens, yaitu penamaan warna berdasarkan fungsi UI seperti Text/Primary, Background/Primary, atau Border/Primary. Token tidak sebaiknya langsung memakai HEX, tetapi dihubungkan ke Colors/Primitives. Setelah token dibuat, tambahkan Modes seperti Light dan Dark. Satu token bisa memiliki nilai berbeda pada mode berbeda. Terakhir, token diterapkan ke UI component. Dengan cara ini, desain bisa berganti Light Mode dan Dark Mode tanpa harus mengubah semua elemen satu per satu.**
